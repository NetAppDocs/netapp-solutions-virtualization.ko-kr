= NetApp ONTAP 스토리지를 사용하는 Proxmox Backup Server의 배포 요구 사항 및 고려 사항
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Proxmox Backup Server(PBS)를 NetApp ONTAP 스토리지와 함께 구현하기 전에 배포 요구 사항 및 고려 사항을 검토하십시오. 이를 통해 백업 서버 배포, 스토리지 구성 및 네트워크 연결에 대한 적절한 계획을 수립할 수 있습니다.



== 배포 시 고려 사항

PBS를 베어메탈에 배포하거나 Proxmox VE의 VM으로 배포할 수 있습니다. PBS를 VM으로 배포할 때는 Proxmox VE 호스트에 다운타임이 발생할 경우를 대비하여 백업 서버의 가용성을 고려해야 합니다. 기본 호스트 장애 발생 시 백업 서버 가용성을 보장하기 위해 PBS를 베어메탈 또는 별도의 Proxmox VE 호스트 또는 클러스터에 배포하십시오.

백업 내용은 데이터 저장소 내에 자체적으로 저장되므로 필요한 경우 다른 PBS 인스턴스로 데이터 저장소를 마이그레이션할 수 있습니다. 이를 통해 원래 PBS 인스턴스를 사용할 수 없는 경우에도 백업 데이터에 계속 액세스할 수 있습니다.



== 스토리지 백엔드 지원

PBS는 로컬 스토리지, NFS, iSCSI, FC, NVMe-oF 및 S3(기술 미리보기)를 포함한 다양한 스토리지 백엔드를 지원합니다. NetApp ONTAP 스토리지와의 통합을 통해 조직은 향상된 데이터 보호 및 복구를 위한 고급 데이터 관리 기능을 활용할 수 있습니다.

재해 복구를 위해 ONTAP SnapMirror 사용하여 PBS 데이터 저장소를 다른 ONTAP 시스템으로 복제하십시오. 이는 백업 데이터를 보호하고 사이트 장애 발생 후 복원을 가능하게 합니다.



== 멀티 클러스터 관리

단일 PBS 인스턴스는 여러 Proxmox VE 클러스터를 관리할 수 있으므로 대규모 배포를 위한 중앙 집중식 백업을 제공합니다. 네임스페이스를 사용하여 서로 다른 클러스터 또는 테넌트의 백업 데이터를 논리적으로 분리함으로써 데이터 격리 및 보안을 보장하십시오.

데이터 보존 정책을 구성하여 백업 데이터의 수명 주기를 자동으로 관리하고 조직 요구 사항을 준수하도록 하십시오. 오래되거나 불필요한 백업 데이터를 자동으로 삭제하도록 정리 정책을 설정하십시오. 가비지 컬렉션 프로세스는 데이터 저장소에서 사용되지 않거나 더 이상 필요 없는 데이터를 제거하여 저장 공간을 확보합니다.



== 관리 인터페이스

웹 기반 인터페이스, 명령줄 도구 또는 REST API를 통해 PBS를 관리할 수 있습니다. URL을 사용하여 8007번 포트의 웹 인터페이스에 접속하세요. `https://<pbs-ip-address>:8007`.



== 네트워크 구성 모범 사례

고가용성과 성능을 위해 PBS와 ONTAP 스토리지 간의 이중화된 네트워크 경로를 확보하십시오. 대역폭 및 이중화 향상을 위해 링크 집합(LACP)을 고려하십시오. 스토리지 트래픽 성능 향상을 위해 모든 네트워크 장치에서 점보 프레임(MTU 9000)을 구성하십시오.

NFS의 경우, 적절한 권한을 부여하여 PBS 데이터 저장소용 전용 내보내기를 생성하십시오. 블록 프로토콜의 경우, 승인된 PBS 호스트만 접근할 수 있도록 적절한 영역 설정 및 LUN 마스킹을 적용해야 합니다.
